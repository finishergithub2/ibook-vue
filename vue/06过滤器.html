# 06过滤器

# 介绍

相同的数据，但表现形式可以有多种形式，

例如

  • 时间可以分为 时间戳 、 格式化、对象 三种形式
  • 字符串可以分为大写、小写、首字母大写等形式

如果原始数据是时间戳格式，想把其变为 格式化 的，解决方法之一是利用“过滤器”

过滤器本质是函数，可以对数据信息进行 二次加工处理 操作

如下时间信息通过 对象 或 时间戳 显示都不合适，但是变为格式化时间就比较友好

Thu Mar 21 2019 17:48:17 GMT+0800     =>   2019-03-21 17:48:17

1553161717825	     									=>  2019-03-21 17:48:17

注意：

​ 过滤器只可以用在两个地方： 插值表达式和 :冒号 属性绑定。

# 全局过滤器

给Vue通过filter方法声明的过滤器是全局的，该Vue的所有实例都可以使用

定义语法：

Vue.filter('过滤器的名称', function(origin){
	// origin 代表被过滤的数据
	// 对数据进行处理
	// 最后 必须 return 一个处理的结果  
})

使用语法：

<span>{{ dt | 过滤器的名称 }}</span>
<span :title="dt | 过滤器的名称"></span>

过滤器与被处理的数据之间通过 竖线| 分隔

案例：

  • 时间过滤器
  • 多vue实例情形(了解)

# 私有过滤器

Vue实例化过程中，与el、data平行的位置声明filters成员并在其中制作过滤器，这个过滤器只能被当前Vue渲染的页面使用，称为 "私有过滤器"

定义语法：

var vm = new Vue({
  filters: {
    // 如下是方法成员，是最新的ES6写法【推荐】
    过滤器的名称(origin){
      return 处理的结果
    }
  }
})

全局 和 私有 过滤器如果同时存在，并且名称还相同， 则按照 就近原则 调用，即私有的优先

全局和私有过滤器取舍：

大多数情况下，全局 和 私有过滤器 都可以使用，但是有的时候，我们并不能直面Vue构造函数(例如后期接触的组件)，这时候就只能乖乖使用 私有过滤器 了

案例：

  • 时间过滤器

# 传递参数

有的时候，过滤器主体功能不变化的情况下，可能结果的形式根据业务要求有所调整，例如同样是时间，有北京时间、巴黎时间等等，地区可以通过给过滤器传递参数实现显示。

过滤器函数 第一个形参是被处理数据，第二、三...形参 接收过滤器传递过来的实参

定义实参：

<span>{{ dt | 过滤器的名称(实参，实参...) }}</span>

定义形参：

Vue.filter('过滤器的名称', function(origin,形参，形参...){
})

# 小结：

  1. 过滤器分为 全局和私有 的两种类型，关键字有filter 和 filters
  2. 过滤器根据业务需要可以传递参数
  3. 只有 插值表达式 和 属性绑定 可以应用过滤器
