# 17组件

# 什么是组件

组件就是有一定功能的多个html标签的集合体 例如 Element-UI组件库中有实现分页、按钮、时间选取器、form表单、table表格等等的许多组件

# 使用组件好处

  • 组件可以被重复调用，提高开发效率
  • 组件可以使得代码编写量减少，提高开发速度
  • 网络上有许多第三方开发好的组件库，可以明显提升开发速度
  • 组件化能够让程序员更专注于业务逻辑

打个比方：之前绘制一个table表格，需要编写table、tr、td一共20个标签，现在通过组件对它们进行汇总，以后就可以通过一个组件标签代表这20个普通的html标签，使得开发速度非常高效

# 私有组件

有些场合，组件不具备通用性，只被在当前vue实例使用，其他场景不使用，那么就可以声明为私有组件

在每个vue实例中，和 el, data, methods, filters 平级的，还可以声明 一个components成员，表示要注册“私有组件”；

语法：

components:{
    '组件的名称': { 组件的配置对象成员 }, 
    '组件的名称': { 组件的配置对象成员 }...
}

例如：

components:{
  'my-page',{
    template:`
      <ul>
        <li>上一页</li>
        <li>[1]</li>
        <li>2</li>
        <li>[3]</li>
        <li>下一页</li>
      </ul>
      `,
  }
}

在组件的 配置对象 中，使用 template 属性指定当前组件要渲染的模板结构

template属性值是多行的，可以通过反单引号圈选编辑

注意：

​ 组件中的DOM结构，有且只能有唯一的根元素（Root Element）来进行包裹(例如上述的ul被当做根元素)

​ 如果有多个根元素，系统只会识别第一个

使用组件：

<my-page></my-page>

# 组件相关成员

可以认为组件是特殊的Vue实例，拥有着与Vue实例相同的成员 例如 组件中可以使用data、methods、filters、directives、computed以及生命周期等成员

例如：

Vue.component('my-page',{
  created(){  // 生命周期函数
    console.log('组件的created被执行了');
  },
  template:`
    <ul>
      <li>{{ prev }}</li>
      <li>[1]</li>
      <li>2</li>
      <li>[3]</li>
      <li @click="clk">{{ next }}</li>
    </ul>
    `,
  data(){   // data成员
    return {prev:'上一页',next:'下一页'}
  },
  methods:{ // methods成员
    clk(){
      console.log('进入下一页');
    }
  }
})

组件 和 Vue实例 的异同：

  1. 组件中的 data 必须是一个 function 并 return 一个 字面量对象 (Vue 实例的 data 一般是 字面量对象，当然与组件一样也可以是 function方法)
  2. 组件中直接通过 template 属性来指定组件的UI结构 Vue 实例中，一般通过 el 属性来指定渲染的容器，当然也可以使用template
  3. 组件和Vue实例类似，都有自己的生命周期函数，过滤器，methods、data等成员

# 全局组件

一个组件需要在同一个Vue的许多实例中被应用，就声明为全局的

定义组件的语法：

Vue.component('组件的名称', { 组件的配置对象成员 })

其他方面与私有组件完全一致

# 小结

  1. 组件是拥有一定功能多个html标签的集合体
  2. 组件根据定义类型分为 全局/私有 组件
  3. 组件是特殊的Vue实例，与Vue实例拥有着基本相同的成员(methods/created/data/template等)
  4. template都必须有一个根元素节点
  5. 组件是为了方便一组功能标签 被重复使用 而引入的，提高开发效率
