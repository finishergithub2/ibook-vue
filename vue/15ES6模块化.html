# 15ES6模块化

# 什么是模块化

模块化 就是一些程序代码经常被其他业务场景使用，为了避免重复开发，就把这些代码设置为共享模式，共享模式就是模块化。

模块化 代码可以是 常量、对象 、函数 等信息 (对象被经常用作模块化)

像jquery、axios等等都是模块化的体现，需要的时候直接拿过来用即可

# commonjs的模块化

导出变量

modules.exports = {name:xxx,height:xxx}

导入变量

const obj = require('a.js')

通过modules.exports 、require() 方式做数据导出、导入功能就是“模块化”的实现，这种模块化称为“CommonJS模块化”

上图的tools.js文件内部有许多变量信息，其他程序文件中如果也想用这些变量，就得自己定义，那么重复工作量很多，解决方法就是“模块化”，其他文件就可以把tools.js文件当做一个模块引入并使用，提到了代码重复利用率，

其他文件使用tools.js模块的过程就是模块化开发

CommonJS模块化本身是民间的(非正式的)，可以在node中运行

(如果要在浏览器端运行可以使用Commonjs的改造版本 CMD和AMD)

官方出品的模块化为ES6模块化，是主流标准，现在暂时还不能在node中运行，(但是后期应该可以)

CommonJS和ES6模块化 都不能在浏览器上直接运行，需要编译后才可以

# 好处

相同的代码被重复使用，提高了代码开发的效率，增强项目的维护性

# 默认导入导出

默认导出：通过一个对象对模块的全部信息做输出处理

默认导出的语法：

export default { name:'jack', height: 170, age:23 }

默认导入的语法：

import 接收名称 from '模块名称'

注意：一个模块中只能用一次默认导出

# 按需导入和导出

按需：一个模块中有多个数据信息，可以根据实际情况只对其中的若干数据做导出操作

导出语法：

export const a = 10
export function square(x){
    return x*x
}

export可以使用多次以便导出多个成员

一般const、function成员用于导出即可，var/let导出没有意义

导入语法：

import { 成员名称,成员名称... } from '模块名称'
import { 成员名称 as 别名,成员名称... } from '模块名称'

导入的时候，可以根据需要导入"一部分"数据进来(不是必须导入全部的)

按需导出的成员，其导入也必须是按需的(默认 和 按需 不能混用)

如果导出的成员名称 与 当前环境名称 有冲突 ，可以通过as设置别名

# 默认和按需导出同时存在

一个模块中， 按需 和 默认 导出情况可以同时存在， 导入时，对号入座即可，默认的 通过 默认方式导入，按需的 通过按需方式导入，当然也可以一并导出

  1. 默认、按需 同时导出语法：

    export const a = 10
export const b = 20
export default {name:'kitty',height:'30'}

    一个模块中默认导出只能出现一次，按需导出可以出现多次

  2. 默认、按需 同时导入语法：

    // a.按需、默认分别各自导入：
import { 成员名称,成员名称... } from '模块名称'   	//只导入按需成员
import obj from '模块名称'						//只导入默认成员
// b.或一并导入：
import obj,{成员名称,成员名称...} from '模块名称'  //按需和默认一并导入

    按需和默认的一并导入，必须是默认的在前、按需的在后

被导入的成员一般只做应用，很少修改，如果非要修改，在语法上支持 数组、对象 成员修改

# 没有导出情形

如果一个模块没有做export导出，那么在应用的地方可以直接做import导入(不用设置名称)

一个模块xxx.js没有做任何导出：

for(var i=0; i<4; i++){

  console.log(i);

}

应用处直接导入即可：

import 'xxx.js'

# 包和模块

包是多个模块的集合

一个包是多个模块的组成，一个模块就是一个js文件

npm install xxx       // 对一个指定包进行安装，安装完毕后，该包里边有许多模块供使用

小结：

  1. 导出，就是共享内容提供方，导出的内容可以被导入者抓取到
  2. 导入，就是共享内容使用者
  3. 默认导出就是通过一个对象把全部的内容都提供出来
  4. 按需导出就是需要哪个信息就导哪个出来，不需要的不操作
  5. 一个模块只能做一次默认导出，但是可以做多次按需导出
