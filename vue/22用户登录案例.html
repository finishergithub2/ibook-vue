# 22用户登录案例

利用 " 路由守卫 + sessionStorage " 实现非登录用户访问后台首页就 强制去登录的效果

上图：前后端分离式项目开发，用户首次登陆系统，服务器端要给前端分配一个token(其是令牌) ，之后前端可以带着token对服务器端进行访问，已便对数据进行 增、删、改、查 操作

token本身在服务器端有时效，过了时间点就要求用户重新登陆并获取新的token

token的使用避免了每次请求输入用户名和密码账号了，token是用户名和密码的变形，但都是是加密信息，更安全

反过来讲，如果前端登录用户没有获取到token，则不允许其登录系统，相反要强制登录

效果：

A. 管理员登录系统后，会生成token令牌信息，自己带着

B.退出系统，销毁token令牌

C. 非登录用户访问后台首页，就会强制退出并让其访问登录页面

小结

  1. 路由 是对 #锚点、onhashchange、hash、component 的封装 使用起来更加方便、高级

  2. 路由本身是一个功能模块，安装后可以使用，应用部分有如下主要步骤：

    1. 导入路由模块import
    2. 注册模块Vue.use()
    3. 导入各个应用组件
    4. 创建对象 path component redirect children
    5. 挂载对象 Vue({router})
    6. router-link
    7. router-view

  3. redirect用于给路由设置重定向

  4. router-link-active是路由切换激活按钮设置css高亮样式

  5. 路由传参：在路由里边通过冒号方式 定义要传递参数，路由里边/子组件中都需要定义props:true

    ​ props:[名称]

  6. 编程式导航：

    ​ router-link是静态方式定义组件切换按钮，是“静态路由导航”

    ​ 有的时候，组件的显示不是通过点击按钮实现，而是在业务逻辑执行过程中根据条件动态加载的，这时候就需要使用“编程式导航”，this.$router.push()/go()/forward()/back(),这是程序代码方式显示的路由跳转

  7. 路由守卫：

    每个路由在执行的时候都必须经过一个关卡，就像旧时代人们要进城必须经过城门一样，在这个关卡中可以对路由做一些业务上的设计，这个关卡就是守卫，即"路由守卫"

    router.beforeEach(function(to,from,next){ next() })

  8. webstorage客户端存储：

    之前在客户端存储数据通常都使用cookie,但是其有种种弊端(例如单个key存储数据量小)，html5发明了webstorage 技术用于替代cookie

    sessionStorage localStorage

    getItem() setItem() removeItem() clear()
